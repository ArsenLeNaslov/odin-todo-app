# Odin's To Do App

A nebula-themed To Do List App that organises tasks across multiple projects. It highlights priorities and overdue items, supports adding and editing To Dos via interactive modals, and saves all data in the browser for persistence.

Referencing Odin's Project: [To Do List App](https://www.theodinproject.com/lessons/node-path-javascript-todo-list)

Live preview: [Odin's To Do App](https://arsenlenaslov.github.io/odin-to-app/)

## Features
- **Create and Manage Projects**: Add multiple projects and switch between them.
- **Add, Edit, and Delete Todos**: Create To Dos with title, description, due date, priority, and optional notes. Edit or remove them at any time.
- **Visual Priority and Status**: To Dos display priority levels (low, medium, high) and completion status with clear visual indicators.
- **Overdue Highlighting**: Automatically marks To Dos as overdue if past their due date.
- **Sample Todos for New Projects**: Automatically adds starter To Dos when a project is empty.
- **Responsive and Interactive UI**: Clickable To Dos open modals for easy editing; projects and To Dos update in real time.
- **Persistent Storage**: All data is saved in the browserâ€™s local storage, so your projects and To Dos remain after a page refresh.
- **Global App Access for Debugging**: Exposes App and Dom objects globally for developers to inspect.
- **Webpack Build**: for bundling and asset management.
- **GitHub Pages Deployment**: Deployed live using the gh-pages branch on GitHub.

## Technologies
- **JavaScript (ES6 Modules & Classes)** for modular code structure, app logic, and DOM manipulation.
- **HTML5** for semantic structure and content rendering.
- **CSS3** for styling, layout, and responsive design.
- **Webpack** for bundling JavaScript modules and assets.
- **Local Storage API** for client-side data persistence.
- **Date-fns** for date formatting and handling overdue/today/tomorrow logic.
- **Event Handling & DOM APIs** for interactive UI elements and modal management.
